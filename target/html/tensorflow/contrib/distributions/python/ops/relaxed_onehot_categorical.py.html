  37 class ExpRelaxedOneHotCategorical(distribution.Distribution):
  38   """ExpRelaxedOneHotCategorical distribution with temperature and logits.
  39 
  40   An ExpRelaxedOneHotCategorical distribution is a log-transformed
  41   RelaxedOneHotCategorical distribution. The RelaxedOneHotCategorical is a
  42   distribution over random probability vectors, vectors of positive real
  43   values that sum to one, which continuously approximates a OneHotCategorical.
  44   The degree of approximation is controlled by a temperature: as the temperature
  45   goes to 0 the RelaxedOneHotCategorical becomes discrete with a distribution
  46   described by the logits, as the temperature goes to infinity the
  47   RelaxedOneHotCategorical becomes the constant distribution that is identically
  48   the constant vector of (1/event_size, ..., 1/event_size).
  49 
  50   Because computing log-probabilities of the RelaxedOneHotCategorical can
  51   suffer from underflow issues, this class is one solution for loss
  52   functions that depend on log-probabilities, such as the KL Divergence found
  53   in the variational autoencoder loss. The KL divergence between two
  54   distributions is invariant under invertible transformations, so evaluating
  55   KL divergences of ExpRelaxedOneHotCategorical samples, which are always
  56   followed by a `tf.exp` op, is equivalent to evaluating KL divergences of
  57   RelaxedOneHotCategorical samples. See the appendix of Maddison et al., 2016
  58   for more mathematical details, where this distribution is called the
  59   ExpConcrete.
  60 
  61   #### Examples
  62 
  63   Creates a continuous distribution, whose exp approximates a 3-class one-hot
  64   categorical distribution. The 2nd class is the most likely to be the
  65   largest component in samples drawn from this distribution. If those samples
  66   are followed by a `tf.exp` op, then they are distributed as a relaxed onehot
  67   categorical.
  68 
  69   ```python
  70   temperature = 0.5
  71   p = [0.1, 0.5, 0.4]
  72   dist = ExpRelaxedOneHotCategorical(temperature, probs=p)
  73   samples = dist.sample()
  74   exp_samples = tf.exp(samples)
  75   # exp_samples has the same distribution as samples from
  76   # RelaxedOneHotCategorical(temperature, probs=p)
  77   ```
  78 
  79   Creates a continuous distribution, whose exp approximates a 3-class one-hot
  80   categorical distribution. The 2nd class is the most likely to be the
  81   largest component in samples drawn from this distribution.
  82 
  83   ```python
  84   temperature = 0.5
  85   logits = [-2, 2, 0]
  86   dist = ExpRelaxedOneHotCategorical(temperature, logits=logits)
  87   samples = dist.sample()
  88   exp_samples = tf.exp(samples)
  89   # exp_samples has the same distribution as samples from
  90   # RelaxedOneHotCategorical(temperature, probs=p)
  91   ```
  92 
  93   Creates a continuous distribution, whose exp approximates a 3-class one-hot
  94   categorical distribution. Because the temperature is very low, samples from
  95   this distribution are almost discrete, with one component almost 0 and the
  96   others very negative. The 2nd class is the most likely to be the largest
  97   component in samples drawn from this distribution.
  98 
  99   ```python
 100   temperature = 1e-5
 101   logits = [-2, 2, 0]
 102   dist = ExpRelaxedOneHotCategorical(temperature, logits=logits)
 103   samples = dist.sample()
 104   exp_samples = tf.exp(samples)
 105   # exp_samples has the same distribution as samples from
 106   # RelaxedOneHotCategorical(temperature, probs=p)
 107   ```
 108 
 109   Creates a continuous distribution, whose exp approximates a 3-class one-hot
 110   categorical distribution. Because the temperature is very high, samples from
 111   this distribution are usually close to the (-log(3), -log(3), -log(3)) vector.
 112   The 2nd class is still the most likely to be the largest component
 113   in samples drawn from this distribution.
 114 
 115   ```python
 116   temperature = 10
 117   logits = [-2, 2, 0]
 118   dist = ExpRelaxedOneHotCategorical(temperature, logits=logits)
 119   samples = dist.sample()
 120   exp_samples = tf.exp(samples)
 121   # exp_samples has the same distribution as samples from
 122   # RelaxedOneHotCategorical(temperature, probs=p)
 123   ```
 124 
 125   Chris J. Maddison, Andriy Mnih, and Yee Whye Teh. The Concrete Distribution:
 126   A Continuous Relaxation of Discrete Random Variables. 2016.
 127   """

 129   @deprecation.deprecated(
 130       "2018-10-01",
 131       "The TensorFlow Distributions library has moved to "
 132       "TensorFlow Probability "
 133       "(https://github.com/tensorflow/probability). You "

 137   def __init__(
 138       self,
 139       temperature,
 140       logits=None,
 141       probs=None,
 142       dtype=None,
 143       validate_args=False,
 144       allow_nan_stats=True,
 145       name="ExpRelaxedOneHotCategorical"):
 146     """Initialize ExpRelaxedOneHotCategorical using class log-probabilities.
 147 
 148     Args:
 149       temperature: An 0-D `Tensor`, representing the temperature
 150         of a set of ExpRelaxedCategorical distributions. The temperature should
 151         be positive.
 152       logits: An N-D `Tensor`, `N >= 1`, representing the log probabilities
 153         of a set of ExpRelaxedCategorical distributions. The first
 154         `N - 1` dimensions index into a batch of independent distributions and
 155         the last dimension represents a vector of logits for each class. Only
 156         one of `logits` or `probs` should be passed in.
 157       probs: An N-D `Tensor`, `N >= 1`, representing the probabilities
 158         of a set of ExpRelaxedCategorical distributions. The first
 159         `N - 1` dimensions index into a batch of independent distributions and
 160         the last dimension represents a vector of probabilities for each
 161         class. Only one of `logits` or `probs` should be passed in.
 162       dtype: The type of the event samples (default: inferred from
 163         logits/probs).
 164       validate_args: Python `bool`, default `False`. When `True` distribution
 165         parameters are checked for validity despite possibly degrading runtime
 166         performance. When `False` invalid inputs may silently render incorrect
 167         outputs.
 168       allow_nan_stats: Python `bool`, default `True`. When `True`, statistics
 169         (e.g., mean, mode, variance) use the value "`NaN`" to indicate the
 170         result is undefined. When `False`, an exception is raised if one or
 171         more of the statistic's batch members are undefined.
 172       name: Python `str` name prefixed to Ops created by this class.
 173     """
 174     parameters = dict(locals())
 175     with ops.name_scope(name, values=[logits, probs, temperature]) as name:
 176 
 177       self._logits, self._probs = distribution_util.get_logits_and_probs(
 178           name=name, logits=logits, probs=probs, validate_args=validate_args,
 179           multidimensional=True)
 180 
 181       if dtype is None:
 182         dtype = self._logits.dtype
 183         if not validate_args:
 184           temperature = math_ops.cast(temperature, dtype)
 185 
 186       with ops.control_dependencies([check_ops.assert_positive(temperature)]
 187                                     if validate_args else []):
 188         self._temperature = array_ops.identity(temperature, name="temperature")
 189         self._temperature_2d = array_ops.reshape(temperature, [-1, 1],
 190                                                  name="temperature_2d")
 191 
 192       logits_shape_static = self._logits.get_shape().with_rank_at_least(1)
 193       if logits_shape_static.ndims is not None:
 194         self._batch_rank = ops.convert_to_tensor(
 195             logits_shape_static.ndims - 1,
 196             dtype=dtypes.int32,
 197             name="batch_rank")
 198       else:
 199         with ops.name_scope(name="batch_rank"):
 200           self._batch_rank = array_ops.rank(self._logits) - 1
 201 
 202       with ops.name_scope(name="event_size"):
 203         self._event_size = array_ops.shape(self._logits)[-1]
 204 
 205     super(ExpRelaxedOneHotCategorical, self).__init__(
 206         dtype=dtype,
 207         reparameterization_type=distribution.FULLY_REPARAMETERIZED,
 208         validate_args=validate_args,
 209         allow_nan_stats=allow_nan_stats,
 210         parameters=parameters,
 211         graph_parents=[self._logits,
 212                        self._probs,
 213                        self._temperature],
 214         name=name)

 236   def _batch_shape_tensor(self):
 237     return array_ops.shape(self._logits)[:-1]
 238 
 239   def _batch_shape(self):
 240     return self.logits.get_shape()[:-1]
 241 
 242   def _event_shape_tensor(self):
 243     return array_ops.shape(self.logits)[-1:]
 244 
 245   def _event_shape(self):
 246     return self.logits.get_shape().with_rank_at_least(1)[-1:]

 248   def _sample_n(self, n, seed=None):
 249     sample_shape = array_ops.concat([[n], array_ops.shape(self.logits)], 0)
 250     logits = self.logits * array_ops.ones(sample_shape, dtype=self.dtype)
 251     logits_2d = array_ops.reshape(logits, [-1, self.event_size])
 252     # Uniform variates must be sampled from the open-interval `(0, 1)` rather
 253     # than `[0, 1)`. To do so, we use `np.finfo(self.dtype.as_numpy_dtype).tiny`
 254     # because it is the smallest, positive, "normal" number. A "normal" number
 255     # is such that the mantissa has an implicit leading 1. Normal, positive
 256     # numbers x, y have the reasonable property that, `x + y >= max(x, y)`. In
 257     # this case, a subnormal number (i.e., np.nextafter) can cause us to sample
 258     # 0.
 259     uniform = random_ops.random_uniform(
 260         shape=array_ops.shape(logits_2d),
 261         minval=np.finfo(self.dtype.as_numpy_dtype).tiny,
 262         maxval=1.,
 263         dtype=self.dtype,
 264         seed=seed)
 265     gumbel = -math_ops.log(-math_ops.log(uniform))
 266     noisy_logits = math_ops.div(gumbel + logits_2d, self._temperature_2d)
 267     samples = nn_ops.log_softmax(noisy_logits)
 268     ret = array_ops.reshape(samples, sample_shape)
 269     return ret
 270 
 271   def _log_prob(self, x):
 272     x = self._assert_valid_sample(x)
 273     # broadcast logits or x if need be.
 274     logits = self.logits
 275     if (not x.get_shape().is_fully_defined() or
 276         not logits.get_shape().is_fully_defined() or
 277         x.get_shape() != logits.get_shape()):
 278       logits = array_ops.ones_like(x, dtype=logits.dtype) * logits
 279       x = array_ops.ones_like(logits, dtype=x.dtype) * x
 280     logits_shape = array_ops.shape(math_ops.reduce_sum(logits, axis=[-1]))
 281     logits_2d = array_ops.reshape(logits, [-1, self.event_size])
 282     x_2d = array_ops.reshape(x, [-1, self.event_size])
 283     # compute the normalization constant
 284     k = math_ops.cast(self.event_size, x.dtype)
 285     log_norm_const = (math_ops.lgamma(k)
 286                       + (k - 1.)
 287                       * math_ops.log(self.temperature))
 288     # compute the unnormalized density
 289     log_softmax = nn_ops.log_softmax(logits_2d - x_2d * self._temperature_2d)
 290     log_unnorm_prob = math_ops.reduce_sum(log_softmax, [-1], keepdims=False)
 291     # combine unnormalized density with normalization constant
 292     log_prob = log_norm_const + log_unnorm_prob
 293     # Reshapes log_prob to be consistent with shape of user-supplied logits
 294     ret = array_ops.reshape(log_prob, logits_shape)
 295     return ret
 296 
 297   def _assert_valid_sample(self, x):
 298     if not self.validate_args:
 299       return x
 300     return control_flow_ops.with_dependencies([
 301         check_ops.assert_non_positive(x),
 302         check_ops.assert_near(
 303             array_ops.zeros([], dtype=self.dtype),
 304             math_ops.reduce_logsumexp(x, axis=[-1])),
 305     ], x)

 308 class RelaxedOneHotCategorical(
 309     transformed_distribution.TransformedDistribution):
 310   """RelaxedOneHotCategorical distribution with temperature and logits.
 311 
 312   The RelaxedOneHotCategorical is a distribution over random probability
 313   vectors, vectors of positive real values that sum to one, which continuously
 314   approximates a OneHotCategorical. The degree of approximation is controlled by
 315   a temperature: as the temperature goes to 0 the RelaxedOneHotCategorical
 316   becomes discrete with a distribution described by the `logits` or `probs`
 317   parameters, as the temperature goes to infinity the RelaxedOneHotCategorical
 318   becomes the constant distribution that is identically the constant vector of
 319   (1/event_size, ..., 1/event_size).
 320 
 321   The RelaxedOneHotCategorical distribution was concurrently introduced as the
 322   Gumbel-Softmax (Jang et al., 2016) and Concrete (Maddison et al., 2016)
 323   distributions for use as a reparameterized continuous approximation to the
 324   `Categorical` one-hot distribution. If you use this distribution, please cite
 325   both papers.
 326 
 327   #### Examples
 328 
 329   Creates a continuous distribution, which approximates a 3-class one-hot
 330   categorical distribution. The 2nd class is the most likely to be the
 331   largest component in samples drawn from this distribution.
 332 
 333   ```python
 334   temperature = 0.5
 335   p = [0.1, 0.5, 0.4]
 336   dist = RelaxedOneHotCategorical(temperature, probs=p)
 337   ```
 338 
 339   Creates a continuous distribution, which approximates a 3-class one-hot
 340   categorical distribution. The 2nd class is the most likely to be the
 341   largest component in samples drawn from this distribution.
 342 
 343   ```python
 344   temperature = 0.5
 345   logits = [-2, 2, 0]
 346   dist = RelaxedOneHotCategorical(temperature, logits=logits)
 347   ```
 348 
 349   Creates a continuous distribution, which approximates a 3-class one-hot
 350   categorical distribution. Because the temperature is very low, samples from
 351   this distribution are almost discrete, with one component almost 1 and the
 352   others nearly 0. The 2nd class is the most likely to be the largest component
 353   in samples drawn from this distribution.
 354 
 355   ```python
 356   temperature = 1e-5
 357   logits = [-2, 2, 0]
 358   dist = RelaxedOneHotCategorical(temperature, logits=logits)
 359   ```
 360 
 361   Creates a continuous distribution, which approximates a 3-class one-hot
 362   categorical distribution. Because the temperature is very high, samples from
 363   this distribution are usually close to the (1/3, 1/3, 1/3) vector. The 2nd
 364   class is still the most likely to be the largest component
 365   in samples drawn from this distribution.
 366 
 367   ```python
 368   temperature = 10
 369   logits = [-2, 2, 0]
 370   dist = RelaxedOneHotCategorical(temperature, logits=logits)
 371   ```
 372 
 373   Eric Jang, Shixiang Gu, and Ben Poole. Categorical Reparameterization with
 374   Gumbel-Softmax. 2016.
 375 
 376   Chris J. Maddison, Andriy Mnih, and Yee Whye Teh. The Concrete Distribution:
 377   A Continuous Relaxation of Discrete Random Variables. 2016.
 378   """
