  15 """Statistical test assertions calibrated for their error rates.
  16 
  17 Statistical tests have an inescapable probability of error: a correct
  18 sampler can still fail a test by chance, and an incorrect sampler can
  19 still pass a test by chance.  This library is about bounding both of
  20 those error rates.  This requires admitting a task-specific notion of
  21 "discrepancy": Correct code will fail rarely, code that misbehaves by
  22 more than the discrepancy will pass rarely, and nothing reliable can
  23 be said about code that misbehaves, but misbehaves by less than the
  24 discrepancy.
  25 
  26 # Example
  27 
  28 Consider testing that the mean of a scalar probability distribution P
  29 is some expected constant.  Suppose the support of P is the interval
  30 `[0, 1]`.  Then you might do this:
  31 
  32 ```python
  33   from tensorflow_probability.python.distributions.internal import statistical_testing
  34 
  35   expected_mean = ...
  36   num_samples = 5000
  37   samples = ... draw 5000 samples from P
  38 
  39   # Check that the mean looks right
  40   check1 = statistical_testing.assert_true_mean_equal_by_dkwm(
  41       samples, low=0., high=1., expected=expected_mean,
  42       false_fail_rate=1e-6)
  43 
  44   # Check that the difference in means detectable with 5000 samples is
  45   # small enough
  46   check2 = tf.assert_less(
  47       statistical_testing.min_discrepancy_of_true_means_detectable_by_dkwm(
  48           num_samples, low=0., high=1.0,
  49           false_fail_rate=1e-6, false_pass_rate=1e-6),
  50       0.01)
  51 
  52   # Be sure to execute both assertion ops
  53   sess.run([check1, check2])
  54 ```
  55 
  56 The second assertion is an instance of experiment design.  It's a
  57 deterministic computation (independent of the code under test) that
  58 checks that `5000` samples is enough to reliably resolve mean
  59 differences of `0.01` or more.  Here "reliably" means that if the code
  60 under test is correct, the probability of drawing an unlucky sample
  61 that causes this test to fail is at most 1e-6; and if the code under
  62 test is incorrect enough that its true mean is 0.01 more or less than
  63 expected, then the probability of drawing a "lucky" sample that causes
  64 the test to false-pass is also at most 1e-6.
  65 
  66 # Overview
  67 
  68 Every function in this library can be characterized in terms of:
  69 
  70 - The property being tested, such as the full density of the
  71   distribution under test, or just its true mean, or a single
  72   Bernoulli probability, etc.
  73 
  74 - The relation being asserted, e.g., whether the mean is less, more,
  75   or equal to the given expected value.
  76 
  77 - The stochastic bound being relied upon, such as the
  78   [Dvoretzky-Kiefer-Wolfowitz-Massart inequality]
  79   (https://en.wikipedia.org/wiki/CDF-based_nonparametric_confidence_interval)
  80   or the CDF of the binomial distribution (for assertions about
  81   Bernoulli probabilities).
  82 
  83 - The number of sample sets in the statistical test.  For example,
  84   testing equality of means has a one-sample variant, where the
  85   expected mean is given exactly, and a two-sample variant, where the
  86   expected mean is itself given by a set of samples (e.g., from an
  87   alternative algorithm).
  88 
  89 - What operation(s) of the test are to be performed.  Each test has
  90   three of these:
  91 
  92   1. `assert` executes the test.  Specifically, it creates a TF op that
  93      produces an error if it has enough evidence to prove that the
  94      property under test is violated.  These functions depend on the
  95      desired false failure rate, because that determines the sizes of
  96      appropriate confidence intervals, etc.
  97 
  98   2. `min_discrepancy` computes the smallest difference reliably
  99      detectable by that test, given the sample count and error rates.
 100      What it's a difference of is test-specific.  For example, a test
 101      for equality of means would make detection guarantees about the
 102      difference the true means.
 103 
 104   3. `min_num_samples` computes the minimum number of samples needed
 105      to reliably detect a given discrepancy with given error rates.
 106 
 107   The latter two are for experimental design, and are meant to be
 108   usable either interactively or inline in the overall test method.
 109 
 110 This library follows a naming convention, to make room for every
 111 combination of the above.  A name mentions the operation first, then
 112 the property, then the relation, then the bound, then, if the test
 113 takes more than one set of samples, a token indicating this.  For
 114 example, `assert_true_mean_equal_by_dkwm` (which is implicitly
 115 one-sample).  Each name is a grammatically sound noun phrase (or verb
 116 phrase, for the asserts).
 117 
 118 # Asymptotic properties
 119 
 120 The number of samples needed tends to scale as `O(1/discrepancy**2)` and
 121 as `O(log(1/error_rate))`.
 122 """

 150 def _batch_sort_vector(x, ascending=True, name=None):
 151   with ops.name_scope(name, "_batch_sort_vector", [x]):
 152     x = ops.convert_to_tensor(x, name="x")
 153     n = array_ops.shape(x)[-1]
 154     if ascending:
 155       y, _ = nn_ops.top_k(-x, k=n, sorted=True)
 156       y = -y
 157     else:
 158       y, _ = nn_ops.top_k(x, k=n, sorted=True)
 159     y.set_shape(x.shape)
 160     return y

 163 def _do_maximum_mean(samples, envelope, high, name=None):
 164   """Common code between maximum_mean and minimum_mean."""
 165   with ops.name_scope(name, "do_maximum_mean", [samples, envelope, high]):
 166     n = array_ops.rank(samples)
 167     # Move the batch dimension of `samples` to the rightmost position,
 168     # where the _batch_sort_vector function wants it.
 169     perm = array_ops.concat([math_ops.range(1, n), [0]], axis=0)
 170     samples = array_ops.transpose(samples, perm)
 171 
 172     samples = _batch_sort_vector(samples)
 173 
 174     # The maximum mean is given by taking `envelope`-worth of
 175     # probability from the smallest samples and moving it to the
 176     # maximum value.  This amounts to:
 177     # - ignoring the smallest k samples, where `k/n < envelope`
 178     # - taking a `1/n - (envelope - k/n)` part of the index k sample
 179     # - taking all the other samples
 180     # - and adding `envelope * high` at the end.
 181     # The following is a vectorized and batched way of computing this.
 182     # `max_mean_contrib` is a mask implementing the previous.
 183     batch_size = array_ops.shape(samples)[-1]
 184     batch_size = math_ops.cast(batch_size, dtype=samples.dtype.base_dtype)
 185     step = 1. / batch_size
 186     cum_steps = step * math_ops.range(
 187         1, batch_size + 1, dtype=samples.dtype.base_dtype)
 188     max_mean_contrib = clip_ops.clip_by_value(
 189         cum_steps - envelope[..., array_ops.newaxis],
 190         clip_value_min=0.,
 191         clip_value_max=step)
 192     return math_ops.reduce_sum(
 193         samples * max_mean_contrib, axis=-1) + envelope * high

 196 def _maximum_mean(samples, envelope, high, name=None):
 197   """Returns a stochastic upper bound on the mean of a scalar distribution.
 198 
 199   The idea is that if the true CDF is within an `eps`-envelope of the
 200   empirical CDF of the samples, and the support is bounded above, then
 201   the mean is bounded above as well.  In symbols,
 202 
 203   ```none
 204   sup_x(|F_n(x) - F(x)|) < eps
 205   ```
 206 
 207   The 0th dimension of `samples` is interpreted as independent and
 208   identically distributed samples.  The remaining dimensions are
 209   broadcast together with `envelope` and `high`, and operated on
 210   separately.
 211 
 212   Args:
 213     samples: Floating-point `Tensor` of samples from the distribution(s)
 214       of interest.  Entries are assumed IID across the 0th dimension.
 215       The other dimensions must broadcast with `envelope` and `high`.
 216     envelope: Floating-point `Tensor` of sizes of admissible CDF
 217       envelopes (i.e., the `eps` above).
 218     high: Floating-point `Tensor` of upper bounds on the distributions'
 219       supports.  `samples <= high`.
 220     name: A name for this operation (optional).
 221 
 222   Returns:
 223     bound: Floating-point `Tensor` of upper bounds on the true means.
 224 
 225   Raises:
 226     InvalidArgumentError: If some `sample` is found to be larger than
 227       the corresponding `high`.
 228   """
 229   with ops.name_scope(name, "maximum_mean", [samples, envelope, high]):
 230     samples = ops.convert_to_tensor(samples, name="samples")
 231     envelope = ops.convert_to_tensor(envelope, name="envelope")
 232     high = ops.convert_to_tensor(high, name="high")
 233 
 234     xmax = math_ops.reduce_max(samples, axis=[0])
 235     msg = "Given sample maximum value exceeds expectations"
 236     check_op = check_ops.assert_less_equal(xmax, high, message=msg)
 237     with ops.control_dependencies([check_op]):
 238       return array_ops.identity(_do_maximum_mean(samples, envelope, high))

 241 def _minimum_mean(samples, envelope, low, name=None):
 242   """Returns a stochastic lower bound on the mean of a scalar distribution.
 243 
 244   The idea is that if the true CDF is within an `eps`-envelope of the
 245   empirical CDF of the samples, and the support is bounded below, then
 246   the mean is bounded below as well.  In symbols,
 247 
 248   ```none
 249   sup_x(|F_n(x) - F(x)|) < eps
 250   ```
 251 
 252   The 0th dimension of `samples` is interpreted as independent and
 253   identically distributed samples.  The remaining dimensions are
 254   broadcast together with `envelope` and `low`, and operated on
 255   separately.
 256 
 257   Args:
 258     samples: Floating-point `Tensor` of samples from the distribution(s)
 259       of interest.  Entries are assumed IID across the 0th dimension.
 260       The other dimensions must broadcast with `envelope` and `low`.
 261     envelope: Floating-point `Tensor` of sizes of admissible CDF
 262       envelopes (i.e., the `eps` above).
 263     low: Floating-point `Tensor` of lower bounds on the distributions'
 264       supports.  `samples >= low`.
 265     name: A name for this operation (optional).
 266 
 267   Returns:
 268     bound: Floating-point `Tensor` of lower bounds on the true means.
 269 
 270   Raises:
 271     InvalidArgumentError: If some `sample` is found to be smaller than
 272       the corresponding `low`.
 273   """
 274   with ops.name_scope(name, "minimum_mean", [samples, envelope, low]):
 275     samples = ops.convert_to_tensor(samples, name="samples")
 276     envelope = ops.convert_to_tensor(envelope, name="envelope")
 277     low = ops.convert_to_tensor(low, name="low")
 278 
 279     xmin = math_ops.reduce_min(samples, axis=[0])
 280     msg = "Given sample minimum value falls below expectations"
 281     check_op = check_ops.assert_greater_equal(xmin, low, message=msg)
 282     with ops.control_dependencies([check_op]):
 283       return - _do_maximum_mean(-samples, envelope, -low)

 286 def _dkwm_cdf_envelope(n, error_rate, name=None):
 287   """Computes the CDF envelope that the DKWM inequality licenses.
 288 
 289   The [Dvoretzky-Kiefer-Wolfowitz-Massart inequality]
 290   (https://en.wikipedia.org/wiki/CDF-based_nonparametric_confidence_interval)
 291   gives a stochastic bound on the distance between the true cumulative
 292   distribution function (CDF) of any distribution and its empirical
 293   CDF.  To wit, for `n` iid samples from any distribution with CDF F,
 294 
 295   ```none
 296   P(sup_x |F_n(x) - F(x)| > eps) < 2exp(-2n eps^2)
 297   ```
 298 
 299   This function computes the envelope size `eps` as a function of the
 300   number of samples `n` and the desired limit on the left-hand
 301   probability above.
 302 
 303   Args:
 304     n: `Tensor` of numbers of samples drawn.
 305     error_rate: Floating-point `Tensor` of admissible rates of mistakes.
 306     name: A name for this operation (optional).
 307 
 308   Returns:
 309     eps: `Tensor` of maximum distances the true CDF can be from the
 310       empirical CDF.  This scales as `O(sqrt(-log(error_rate)))` and
 311       as `O(1 / sqrt(n))`.  The shape is the broadcast of `n` and
 312       `error_rate`.
 313   """
 314   with ops.name_scope(name, "dkwm_cdf_envelope", [n, error_rate]):
 315     n = math_ops.cast(n, dtype=error_rate.dtype)
 316     return math_ops.sqrt(-gen_math_ops.log(error_rate / 2.) / (2. * n))

 319 def _check_shape_dominates(samples, parameters):
 320   """Check that broadcasting `samples` against `parameters` does not expand it.
 321 
 322   Why?  Because I want to be very sure that the samples tensor is not
 323   accidentally enlarged by broadcasting against tensors that are
 324   supposed to be describing the distribution(s) sampled from, lest the
 325   sample counts end up inflated.
 326 
 327   Args:
 328     samples: A `Tensor` whose shape is to be protected against broadcasting.
 329     parameters: A list of `Tensor`s who are parameters for the statistical test.
 330 
 331   Returns:
 332     samples: Return original `samples` with control dependencies attached
 333       to ensure no broadcasting.
 334   """
 335   def check(t):
 336     samples_batch_shape = array_ops.shape(samples)[1:]
 337     broadcasted_batch_shape = array_ops.broadcast_dynamic_shape(
 338         samples_batch_shape, array_ops.shape(t))
 339     # This rank check ensures that I don't get a wrong answer from the
 340     # _shapes_ broadcasting against each other.
 341     samples_batch_ndims = array_ops.size(samples_batch_shape)
 342     ge = check_ops.assert_greater_equal(
 343         samples_batch_ndims, array_ops.rank(t))
 344     eq = check_ops.assert_equal(samples_batch_shape, broadcasted_batch_shape)
 345     return ge, eq
 346   checks = list(itertools.chain(*[check(t) for t in parameters]))
 347   with ops.control_dependencies(checks):
 348     return array_ops.identity(samples)

 351 def true_mean_confidence_interval_by_dkwm(
 352     samples, low, high, error_rate=1e-6, name=None):
 353   """Computes a confidence interval for the mean of a scalar distribution.
 354 
 355   In batch mode, computes confidence intervals for all distributions
 356   in the batch (which need not be identically distributed).
 357 
 358   Relies on the [Dvoretzky-Kiefer-Wolfowitz-Massart inequality]
 359   (https://en.wikipedia.org/wiki/CDF-based_nonparametric_confidence_interval).
 360 
 361   The probability (over the randomness of drawing the given samples)
 362   that any true mean is outside the corresponding returned interval is
 363   no more than the given `error_rate`.  The size of the intervals
 364   scale as
 365   `O(1 / sqrt(#samples))`, as `O(high - low)`, and as `O(-log(error_rate))`.
 366 
 367   Note that `error_rate` is a total error rate for all the confidence
 368   intervals in the batch.  As such, if the batch is nontrivial, the
 369   error rate is not broadcast but divided (evenly) among the batch
 370   members.
 371 
 372   Args:
 373     samples: Floating-point `Tensor` of samples from the distribution(s)
 374       of interest.  Entries are assumed IID across the 0th dimension.
 375       The other dimensions must broadcast with `low` and `high`.
 376       The support is bounded: `low <= samples <= high`.
 377     low: Floating-point `Tensor` of lower bounds on the distributions'
 378       supports.
 379     high: Floating-point `Tensor` of upper bounds on the distributions'
 380       supports.
 381     error_rate: *Scalar* floating-point `Tensor` admissible total rate
 382       of mistakes.
 383     name: A name for this operation (optional).
 384 
 385   Returns:
 386     low: A floating-point `Tensor` of stochastic lower bounds on the
 387       true means.
 388     high: A floating-point `Tensor` of stochastic upper bounds on the
 389       true means.
 390   """
 391   with ops.name_scope(
 392       name, "true_mean_confidence_interval_by_dkwm",
 393       [samples, low, high, error_rate]):
 394     samples = ops.convert_to_tensor(samples, name="samples")
 395     low = ops.convert_to_tensor(low, name="low")
 396     high = ops.convert_to_tensor(high, name="high")
 397     error_rate = ops.convert_to_tensor(error_rate, name="error_rate")
 398     samples = _check_shape_dominates(samples, [low, high])
 399     check_ops.assert_scalar(error_rate)  # Static shape
 400     error_rate = _itemwise_error_rate(error_rate, [low, high], samples)
 401     n = array_ops.shape(samples)[0]
 402     envelope = _dkwm_cdf_envelope(n, error_rate)
 403     min_mean = _minimum_mean(samples, envelope, low)
 404     max_mean = _maximum_mean(samples, envelope, high)
 405     return min_mean, max_mean

 408 def _itemwise_error_rate(
 409     total_error_rate, param_tensors, sample_tensor=None, name=None):
 410   with ops.name_scope(
 411       name, "itemwise_error_rate",
 412       [total_error_rate, param_tensors, sample_tensor]):
 413     result_shape = [1]
 414     for p_tensor in param_tensors:
 415       result_shape = array_ops.broadcast_dynamic_shape(
 416           array_ops.shape(p_tensor), result_shape)
 417     if sample_tensor is not None:
 418       result_shape = array_ops.broadcast_dynamic_shape(
 419           array_ops.shape(sample_tensor)[1:], result_shape)
 420     num_items = math_ops.reduce_prod(result_shape)
 421     return total_error_rate / math_ops.cast(
 422         num_items, dtype=total_error_rate.dtype)

 425 def assert_true_mean_equal_by_dkwm(
 426     samples, low, high, expected, false_fail_rate=1e-6, name=None):
 427   """Asserts the mean of the given distribution is as expected.
 428 
 429   More precisely, fails if there is enough evidence (using the
 430   [Dvoretzky-Kiefer-Wolfowitz-Massart inequality]
 431   (https://en.wikipedia.org/wiki/CDF-based_nonparametric_confidence_interval))
 432   that the true mean of some distribution from which the given samples are
 433   drawn is _not_ the given expected mean with statistical significance
 434   `false_fail_rate` or stronger, otherwise passes.  If you also want to
 435   check that you are gathering enough evidence that a pass is not
 436   spurious, see `min_num_samples_for_dkwm_mean_test` and
 437   `min_discrepancy_of_true_means_detectable_by_dkwm`.
 438 
 439   Note that `false_fail_rate` is a total false failure rate for all
 440   the assertions in the batch.  As such, if the batch is nontrivial,
 441   the assertion will insist on stronger evidence to fail any one member.
 442 
 443   Args:
 444     samples: Floating-point `Tensor` of samples from the distribution(s)
 445       of interest.  Entries are assumed IID across the 0th dimension.
 446       The other dimensions must broadcast with `low` and `high`.
 447       The support is bounded: `low <= samples <= high`.
 448     low: Floating-point `Tensor` of lower bounds on the distributions'
 449       supports.
 450     high: Floating-point `Tensor` of upper bounds on the distributions'
 451       supports.
 452     expected: Floating-point `Tensor` of expected true means.
 453     false_fail_rate: *Scalar* floating-point `Tensor` admissible total
 454       rate of mistakes.
 455     name: A name for this operation (optional).
 456 
 457   Returns:
 458     check: Op that raises `InvalidArgumentError` if any expected mean is
 459       outside the corresponding confidence interval.
 460   """
 461   with ops.name_scope(
 462       name, "assert_true_mean_equal_by_dkwm",
 463       [samples, low, high, expected, false_fail_rate]):
 464     return assert_true_mean_in_interval_by_dkwm(
 465         samples, low, high, expected, expected, false_fail_rate)
 466

 468 def min_discrepancy_of_true_means_detectable_by_dkwm(
 469     n, low, high, false_fail_rate, false_pass_rate, name=None):
 470   """Returns the minimum mean discrepancy that a DKWM-based test can detect.
 471 
 472   DKWM is the [Dvoretzky-Kiefer-Wolfowitz-Massart inequality]
 473   (https://en.wikipedia.org/wiki/CDF-based_nonparametric_confidence_interval).
 474 
 475   Note that `false_fail_rate` is a total false failure rate for all
 476   the tests in the batch.  As such, if the batch is nontrivial, each
 477   member will demand more samples.  The `false_pass_rate` is also
 478   interpreted as a total, but is treated asymmetrically: If each test
 479   in the batch detects its corresponding discrepancy with probability
 480   at least `1 - false_pass_rate`, then running all those tests and
 481   failing if any one fails will jointly detect all those discrepancies
 482   with the same `false_pass_rate`.
 483 
 484   Args:
 485     n: `Tensor` of numbers of samples to be drawn from the distributions
 486       of interest.
 487     low: Floating-point `Tensor` of lower bounds on the distributions'
 488       supports.
 489     high: Floating-point `Tensor` of upper bounds on the distributions'
 490       supports.
 491     false_fail_rate: *Scalar* floating-point `Tensor` admissible total
 492       rate of false failures.
 493     false_pass_rate: *Scalar* floating-point `Tensor` admissible rate
 494       of false passes.
 495     name: A name for this operation (optional).
 496 
 497   Returns:
 498     discr: `Tensor` of lower bounds on the distances between true
 499        means detectable by a DKWM-based test.
 500 
 501   For each batch member `i`, of `K` total, drawing `n[i]` samples from
 502   some scalar distribution supported on `[low[i], high[i]]` is enough
 503   to detect a difference in means of size `discr[i]` or more.
 504   Specifically, we guarantee that (a) if the true mean is the expected
 505   mean (resp. in the expected interval), then `assert_true_mean_equal_by_dkwm`
 506   (resp. `assert_true_mean_in_interval_by_dkwm`) will fail with
 507   probability at most `false_fail_rate / K` (which amounts to
 508   `false_fail_rate` if applied to the whole batch at once), and (b) if
 509   the true mean differs from the expected mean (resp. falls outside
 510   the expected interval) by at least `discr[i]`,
 511   `assert_true_mean_equal_by_dkwm`
 512   (resp. `assert_true_mean_in_interval_by_dkwm`) will pass with
 513   probability at most `false_pass_rate`.
 514 
 515   The detectable discrepancy scales as
 516 
 517   - `O(high[i] - low[i])`,
 518   - `O(1 / sqrt(n[i]))`,
 519   - `O(-log(false_fail_rate/K))`, and
 520   - `O(-log(false_pass_rate))`.
 521   """
 522   with ops.name_scope(
 523       name, "min_discrepancy_of_true_means_detectable_by_dkwm",
 524       [n, low, high, false_fail_rate, false_pass_rate]):
 525     n = ops.convert_to_tensor(n, name="n")
 526     low = ops.convert_to_tensor(low, name="low")
 527     high = ops.convert_to_tensor(high, name="high")
 528     false_fail_rate = ops.convert_to_tensor(
 529         false_fail_rate, name="false_fail_rate")
 530     false_pass_rate = ops.convert_to_tensor(
 531         false_pass_rate, name="false_pass_rate")
 532     # Algorithm: Assume a true CDF F.  The DKWM inequality gives a
 533     # stochastic bound on how far the observed empirical CDF F_n can be.
 534     # Then, using the DKWM inequality again gives a stochastic bound on
 535     # the farthest candidate true CDF F' that
 536     # true_mean_confidence_interval_by_dkwm might consider.  At worst, these
 537     # errors may go in the same direction, so the distance between F and
 538     # F' is bounded by the sum.
 539     # On batching: false fail rates sum, so I need to reduce
 540     # the input to account for the batching.  False pass rates
 541     # max, so I don't.
 542     sampling_envelope = _dkwm_cdf_envelope(n, false_pass_rate)
 543     false_fail_rate = _itemwise_error_rate(false_fail_rate, [n, low, high])
 544     analysis_envelope = _dkwm_cdf_envelope(n, false_fail_rate)
 545     return (high - low) * (sampling_envelope + analysis_envelope)

 548 def min_num_samples_for_dkwm_mean_test(
 549     discrepancy, low, high,
 550     false_fail_rate=1e-6, false_pass_rate=1e-6, name=None):
 551   """Returns how many samples suffice for a one-sample DKWM mean test.
 552 
 553   To wit, returns an upper bound on the number of samples necessary to
 554   guarantee detecting a mean difference of at least the given
 555   `discrepancy`, with the given `false_fail_rate` and `false_pass_rate`,
 556   using the [Dvoretzky-Kiefer-Wolfowitz-Massart inequality]
 557   (https://en.wikipedia.org/wiki/CDF-based_nonparametric_confidence_interval)
 558   on a scalar distribution supported on `[low, high]`.
 559 
 560   Args:
 561     discrepancy: Floating-point `Tensor` of desired upper limits on mean
 562       differences that may go undetected with probability higher than
 563       `1 - false_pass_rate`.
 564     low: `Tensor` of lower bounds on the distributions' support.
 565     high: `Tensor` of upper bounds on the distributions' support.
 566     false_fail_rate: *Scalar* floating-point `Tensor` admissible total
 567       rate of false failures.
 568     false_pass_rate: *Scalar* floating-point `Tensor` admissible rate
 569       of false passes.
 570     name: A name for this operation (optional).
 571 
 572   Returns:
 573     n: `Tensor` of numbers of samples to be drawn from the distributions
 574       of interest.
 575 
 576   The `discrepancy`, `low`, and `high` tensors must have
 577   broadcast-compatible shapes.
 578 
 579   For each batch member `i`, of `K` total, drawing `n[i]` samples from
 580   some scalar distribution supported on `[low[i], high[i]]` is enough
 581   to detect a difference in means of size `discrepancy[i]` or more.
 582   Specifically, we guarantee that (a) if the true mean is the expected
 583   mean (resp. in the expected interval), then `assert_true_mean_equal_by_dkwm`
 584   (resp. `assert_true_mean_in_interval_by_dkwm`) will fail with
 585   probability at most `false_fail_rate / K` (which amounts to
 586   `false_fail_rate` if applied to the whole batch at once), and (b) if
 587   the true mean differs from the expected mean (resp. falls outside
 588   the expected interval) by at least `discrepancy[i]`,
 589   `assert_true_mean_equal_by_dkwm`
 590   (resp. `assert_true_mean_in_interval_by_dkwm`) will pass with
 591   probability at most `false_pass_rate`.
 592 
 593   The required number of samples scales
 594   as `O((high[i] - low[i])**2)`, `O(-log(false_fail_rate/K))`,
 595   `O(-log(false_pass_rate))`, and `O(1 / discrepancy[i]**2)`.
 596   """
 597   with ops.name_scope(
 598       name, "min_num_samples_for_dkwm_mean_test",
 599       [low, high, false_fail_rate, false_pass_rate, discrepancy]):
 600     discrepancy = ops.convert_to_tensor(
 601         discrepancy, name="discrepancy")
 602     low = ops.convert_to_tensor(low, name="low")
 603     high = ops.convert_to_tensor(high, name="high")
 604     false_fail_rate = ops.convert_to_tensor(
 605         false_fail_rate, name="false_fail_rate")
 606     false_pass_rate = ops.convert_to_tensor(
 607         false_pass_rate, name="false_pass_rate")
 608     # Could choose to cleverly allocate envelopes, but this is sound.
 609     envelope1 = discrepancy / (2. * (high - low))
 610     envelope2 = envelope1
 611     false_fail_rate = _itemwise_error_rate(
 612         false_fail_rate, [low, high, discrepancy])
 613     n1 = -math_ops.log(false_fail_rate / 2.) / (2. * envelope1**2)
 614     n2 = -math_ops.log(false_pass_rate / 2.) / (2. * envelope2**2)
 615     return math_ops.maximum(n1, n2)
 616

 618 def assert_true_mean_in_interval_by_dkwm(
 619     samples, low, high, expected_low, expected_high,
 620     false_fail_rate=1e-6, name=None):
 621   """Asserts the mean of the given distribution is in the given interval.
 622 
 623   More precisely, fails if there is enough evidence (using the
 624   [Dvoretzky-Kiefer-Wolfowitz-Massart inequality]
 625   (https://en.wikipedia.org/wiki/CDF-based_nonparametric_confidence_interval))
 626   that the mean of the distribution from which the given samples are
 627   drawn is _outside_ the given interval with statistical significance
 628   `false_fail_rate` or stronger, otherwise passes.  If you also want
 629   to check that you are gathering enough evidence that a pass is not
 630   spurious, see `min_num_samples_for_dkwm_mean_test` and
 631   `min_discrepancy_of_true_means_detectable_by_dkwm`.
 632 
 633   Note that `false_fail_rate` is a total false failure rate for all
 634   the assertions in the batch.  As such, if the batch is nontrivial,
 635   the assertion will insist on stronger evidence to fail any one member.
 636 
 637   Args:
 638     samples: Floating-point `Tensor` of samples from the distribution(s)
 639       of interest.  Entries are assumed IID across the 0th dimension.
 640       The other dimensions must broadcast with `low` and `high`.
 641       The support is bounded: `low <= samples <= high`.
 642     low: Floating-point `Tensor` of lower bounds on the distributions'
 643       supports.
 644     high: Floating-point `Tensor` of upper bounds on the distributions'
 645       supports.
 646     expected_low: Floating-point `Tensor` of lower bounds on the
 647       expected true means.
 648     expected_high: Floating-point `Tensor` of upper bounds on the
 649       expected true means.
 650     false_fail_rate: *Scalar* floating-point `Tensor` admissible total
 651       rate of mistakes.
 652     name: A name for this operation (optional).
 653 
 654   Returns:
 655     check: Op that raises `InvalidArgumentError` if any expected mean
 656       interval does not overlap with the corresponding confidence
 657       interval.
 658   """
 659   with ops.name_scope(
 660       name, "assert_true_mean_in_interval_by_dkwm",
 661       [samples, low, high, expected_low, expected_high, false_fail_rate]):
 662     samples = ops.convert_to_tensor(samples, name="samples")
 663     low = ops.convert_to_tensor(low, name="low")
 664     high = ops.convert_to_tensor(high, name="high")
 665     expected_low = ops.convert_to_tensor(expected_low, name="expected_low")
 666     expected_high = ops.convert_to_tensor(expected_high, name="expected_high")
 667     false_fail_rate = ops.convert_to_tensor(
 668         false_fail_rate, name="false_fail_rate")
 669     samples = _check_shape_dominates(
 670         samples, [low, high, expected_low, expected_high])
 671     min_mean, max_mean = true_mean_confidence_interval_by_dkwm(
 672         samples, low, high, false_fail_rate)
 673     # Assert that the interval [min_mean, max_mean] intersects the
 674     # interval [expected_low, expected_high].  This is true if
 675     #   max_mean >= expected_low and min_mean <= expected_high.
 676     # By DeMorgan's law, that's also equivalent to
 677     #   not (max_mean < expected_low or min_mean > expected_high),
 678     # which is a way of saying the two intervals are not disjoint.
 679     check_confidence_interval_can_intersect = check_ops.assert_greater_equal(
 680         max_mean, expected_low, message="Confidence interval does not "
 681         "intersect: true mean smaller than expected")
 682     with ops.control_dependencies([check_confidence_interval_can_intersect]):
 683       return check_ops.assert_less_equal(
 684           min_mean, expected_high, message="Confidence interval does not "
 685           "intersect: true mean greater than expected")
 686

 688 def assert_true_mean_equal_by_dkwm_two_sample(
 689     samples1, low1, high1, samples2, low2, high2,
 690     false_fail_rate=1e-6, name=None):
 691   """Asserts the means of the given distributions are equal.
 692 
 693   More precisely, fails if there is enough evidence (using the
 694   [Dvoretzky-Kiefer-Wolfowitz-Massart inequality]
 695   (https://en.wikipedia.org/wiki/CDF-based_nonparametric_confidence_interval))
 696   that the means of the distributions from which the given samples are
 697   drawn are _not_ equal with statistical significance `false_fail_rate`
 698   or stronger, otherwise passes.  If you also want to check that you
 699   are gathering enough evidence that a pass is not spurious, see
 700   `min_num_samples_for_dkwm_mean_two_sample_test` and
 701   `min_discrepancy_of_true_means_detectable_by_dkwm_two_sample`.
 702 
 703   Note that `false_fail_rate` is a total false failure rate for all
 704   the assertions in the batch.  As such, if the batch is nontrivial,
 705   the assertion will insist on stronger evidence to fail any one member.
 706 
 707   Args:
 708     samples1: Floating-point `Tensor` of samples from the
 709       distribution(s) A.  Entries are assumed IID across the 0th
 710       dimension.  The other dimensions must broadcast with `low1`,
 711       `high1`, `low2`, and `high2`.
 712       The support is bounded: `low1 <= samples1 <= high1`.
 713     low1: Floating-point `Tensor` of lower bounds on the supports of the
 714       distributions A.
 715     high1: Floating-point `Tensor` of upper bounds on the supports of
 716       the distributions A.
 717     samples2: Floating-point `Tensor` of samples from the
 718       distribution(s) B.  Entries are assumed IID across the 0th
 719       dimension.  The other dimensions must broadcast with `low1`,
 720       `high1`, `low2`, and `high2`.
 721       The support is bounded: `low2 <= samples2 <= high2`.
 722     low2: Floating-point `Tensor` of lower bounds on the supports of the
 723       distributions B.
 724     high2: Floating-point `Tensor` of upper bounds on the supports of
 725       the distributions B.
 726     false_fail_rate: *Scalar* floating-point `Tensor` admissible total
 727       rate of mistakes.
 728     name: A name for this operation (optional).
 729 
 730   Returns:
 731     check: Op that raises `InvalidArgumentError` if any pair of confidence
 732       intervals true for corresponding true means do not overlap.
 733   """
 734   with ops.name_scope(
 735       name, "assert_true_mean_equal_by_dkwm_two_sample",
 736       [samples1, low1, high1, samples2, low2, high2, false_fail_rate]):
 737     samples1 = ops.convert_to_tensor(samples1, name="samples1")
 738     low1 = ops.convert_to_tensor(low1, name="low1")
 739     high1 = ops.convert_to_tensor(high1, name="high1")
 740     samples2 = ops.convert_to_tensor(samples2, name="samples2")
 741     low2 = ops.convert_to_tensor(low2, name="low2")
 742     high2 = ops.convert_to_tensor(high2, name="high2")
 743     false_fail_rate = ops.convert_to_tensor(
 744         false_fail_rate, name="false_fail_rate")
 745     samples1 = _check_shape_dominates(samples1, [low1, high1])
 746     samples2 = _check_shape_dominates(samples2, [low2, high2])
 747     compatible_samples = check_ops.assert_equal(
 748         array_ops.shape(samples1)[1:], array_ops.shape(samples2)[1:])
 749     with ops.control_dependencies([compatible_samples]):
 750       # Could in principle play games with cleverly allocating
 751       # significance instead of the even split below.  It may be possible
 752       # to get tighter intervals, in order to obtain a higher power test.
 753       # Any allocation strategy that depends only on the support bounds
 754       # and sample counts should be valid; however, because the intervals
 755       # scale as O(-log(false_fail_rate)), there doesn't seem to be much
 756       # room to win.
 757       min_mean_2, max_mean_2 = true_mean_confidence_interval_by_dkwm(
 758           samples2, low2, high2, false_fail_rate / 2.)
 759       return assert_true_mean_in_interval_by_dkwm(
 760           samples1, low1, high1, min_mean_2, max_mean_2, false_fail_rate / 2.)

 763 def min_discrepancy_of_true_means_detectable_by_dkwm_two_sample(
 764     n1, low1, high1, n2, low2, high2,
 765     false_fail_rate, false_pass_rate, name=None):
 766   """Returns the minimum mean discrepancy for a two-sample DKWM-based test.
 767 
 768   DKWM is the [Dvoretzky-Kiefer-Wolfowitz-Massart inequality]
 769   (https://en.wikipedia.org/wiki/CDF-based_nonparametric_confidence_interval).
 770 
 771   Note that `false_fail_rate` is a total false failure rate for all
 772   the tests in the batch.  As such, if the batch is nontrivial, each
 773   member will demand more samples.  The `false_pass_rate` is also
 774   interpreted as a total, but is treated asymmetrically: If each test
 775   in the batch detects its corresponding discrepancy with probability
 776   at least `1 - false_pass_rate`, then running all those tests and
 777   failing if any one fails will jointly detect all those discrepancies
 778   with the same `false_pass_rate`.
 779 
 780   Args:
 781     n1: `Tensor` of numbers of samples to be drawn from the distributions A.
 782     low1: Floating-point `Tensor` of lower bounds on the supports of the
 783       distributions A.
 784     high1: Floating-point `Tensor` of upper bounds on the supports of
 785       the distributions A.
 786     n2: `Tensor` of numbers of samples to be drawn from the distributions B.
 787     low2: Floating-point `Tensor` of lower bounds on the supports of the
 788       distributions B.
 789     high2: Floating-point `Tensor` of upper bounds on the supports of
 790       the distributions B.
 791     false_fail_rate: *Scalar* floating-point `Tensor` admissible total
 792       rate of false failures.
 793     false_pass_rate: *Scalar* floating-point `Tensor` admissible rate
 794       of false passes.
 795     name: A name for this operation (optional).
 796 
 797   Returns:
 798     discr: `Tensor` of lower bounds on the distances between true means
 799        detectable by a two-sample DKWM-based test.
 800 
 801   For each batch member `i`, of `K` total, drawing `n1[i]` samples
 802   from scalar distribution A supported on `[low1[i], high1[i]]` and `n2[i]`
 803   samples from scalar distribution B supported on `[low2[i], high2[i]]`
 804   is enough to detect a difference in their true means of size
 805   `discr[i]` or more.  Specifically, we guarantee that (a) if their
 806   true means are equal, `assert_true_mean_equal_by_dkwm_two_sample`
 807   will fail with probability at most `false_fail_rate/K` (which
 808   amounts to `false_fail_rate` if applied to the whole batch at once),
 809   and (b) if their true means differ by at least `discr[i]`,
 810   `assert_true_mean_equal_by_dkwm_two_sample` will pass with
 811   probability at most `false_pass_rate`.
 812 
 813   The detectable distribution scales as
 814 
 815   - `O(high1[i] - low1[i])`, `O(high2[i] - low2[i])`,
 816   - `O(1 / sqrt(n1[i]))`, `O(1 / sqrt(n2[i]))`,
 817   - `O(-log(false_fail_rate/K))`, and
 818   - `O(-log(false_pass_rate))`.
 819   """
 820   with ops.name_scope(
 821       name, "min_discrepancy_of_true_means_detectable_by_dkwm_two_sample",
 822       [n1, low1, high1, n2, low2, high2, false_fail_rate, false_pass_rate]):
 823     n1 = ops.convert_to_tensor(n1, name="n1")
 824     low1 = ops.convert_to_tensor(low1, name="low1")
 825     high1 = ops.convert_to_tensor(high1, name="high1")
 826     n2 = ops.convert_to_tensor(n2, name="n2")
 827     low2 = ops.convert_to_tensor(low2, name="low2")
 828     high2 = ops.convert_to_tensor(high2, name="high2")
 829     false_fail_rate = ops.convert_to_tensor(
 830         false_fail_rate, name="false_fail_rate")
 831     false_pass_rate = ops.convert_to_tensor(
 832         false_pass_rate, name="false_pass_rate")
 833     det_disc1 = min_discrepancy_of_true_means_detectable_by_dkwm(
 834         n1, low1, high1, false_fail_rate / 2., false_pass_rate / 2.)
 835     det_disc2 = min_discrepancy_of_true_means_detectable_by_dkwm(
 836         n2, low2, high2, false_fail_rate / 2., false_pass_rate / 2.)
 837     return det_disc1 + det_disc2
 838
